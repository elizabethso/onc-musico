Health eDecisions (HeD)

Initiative Description: Clinical Decision Support (CDS) is the user facing representation of clinical guidance. Effective CDS interventions require availability of computable biomedical knowledge, person-specific data, and a reasoning or inference mechanism. In order to optimize these benefits, CDS interventions must be made more easily shareable and implementable so that any organization can easily acquire and deploy CDS interventions. To this end HeD is creating 2 separate use cases. Use Case 1CDS Artifact Sharing and Use Case 2 CDS Guidance Services. The goal of Use Case 1 is to produce, consume and where feasible, execute implementable CDS interventions which include: Event Condition Action Rules (ECA Rules), Order Sets, Documentation Templates. HeD is in the process of evaluating the standards to support UC 2. Use Case 2 relies on Standards that define how a system can interact with and utilize an electronic interface that provides helpful, actionable clinical guidance.

Meaningful Use Alignment

  • Improve Quality
    • Consume external CDS (SGRP113)
    • CDS for medication allergies (SGRP107)
  • Population Health
    • Immunization CDS (SGRP107)
Standards in use
  • Use Case 1
    • Standards & Data Model Utilized/Referenced to Create Schema
      • Arden Syntax
      • vMR
      • HL7 Order Sets
      • Open EHR
      • CDSC L3
    • Standards & Data Models Evaluated to Ensure Cross Initiative Alignment (this evaluation led to eventual changes to the schema)
      • HQMF
      • QDM
      • FHIM
